## 1. 사업 개요

### 1-1. 피그건 (PigGun) 사업 구조

| 구분 | 개인사업자 | 법인 |
|------|-----------|------|
| 주요 사업 | 오프라인 카페 (커피/음료 판매) | 에어소프트건 체험 서비스 + 중고 매매 |
| 임대료 | 카페 임대료 | 총기 보관 창고(사무실) 임대료 |
| 카드단말기 | 개인 카드단말기 | 법인 카드단말기 |
| 기타 | - | 네이버 카페 광고 수익 (배너 광고비) |

공통사항: 직원 2명 + 아르바이트생, 중고 매입(당근마켓/중고나라, 구매 리스트 작성), 일부 현금 거래 존재

## 2. 기획 배경 (통화 내용 기반)

박종렬 대표님과의 통화에서 도출된 핵심 문제:

- **중고 매입이 대부분 → 개인 간 거래라 매입 부가세가 안 잡힘**
- 매출은 있는데 비용 증빙이 부족해 부가세·법인세 부담이 큼
- 이전에 엑셀 양식을 만들어줬지만 **복잡해서 안 씀** → "심플해야 한다"
- 동생 성격상 **매일 기록은 못하고, 월 1회 정산이 현실적**
- 세금을 "얼마 내고 싶다"에서 역산하는 방식이 동생에게 맞음
- 법인과 개인 **두 개의 사업자를 동시에 관리**해야 함

## 3. 핵심 설계 철학

### 3-1. "역산형" 세무장부
> "야 너 이달에 2천만 원 팔았는데 너 세금 얼마 냈으면 좋겠냐?"
> → 희망 납부세액을 입력하면 **필요한 비용 자료가 자동 역산**

일반적인 장부: 매출 - 매입 - 비용 = 이익 → 세금 계산
**이 장부**: 매출 입력 → 희망세액 입력 → **필요 비용 총액 역산** → 현재 비용과 부족분 표시

### 3-2. 극단적 심플함
- 카드 전표 출력 → 금액만 기입하는 수준
- 월 1회 정리가 기본 주기
- 고정비는 연초 1회 설정 → 매월 자동

### 3-3. 법인 + 개인 탭 분리
- 카드단말기가 분리되어 있으므로 매출 입력도 분리
- 각각 독립된 비용 구조 + 세금 역산

### 4-4. 중고매입 관리 기능 (신규)

> 동생이 이미 당근/중고나라 매입 시 **구매 리스트를 작성**하고 있음 → 이걸 활용

| 입력 항목 | 설명 |
|-----------|------|
| 매입일 | 구매 날짜 |
| 품목명 | 에어소프트건 모델명 등 |
| 매입처 | 당근마켓 / 중고나라 / 직접매입 |
| 매입금액 | 구매 금액 |
| 결제방식 | 당근페이 / 계좌이체 / 현금 |
| 증빙유무 | 있음(세금계산서) / 없음(개인거래) / 간이영수증 |
| 비고 | 메모 |

- 월별 중고매입 합계가 자동으로 비용에 반영
- **증빙 없는 매입 비율**을 경고 표시 → 부가세 공제 못 받는 금액 시각화
- 기존에 작성하던 구매 리스트 습관을 그대로 디지털화

### 4-5. 고정비 자동 입력
- 설정 화면에서 1회 입력 → 12개월 자동 반영
- 개인/법인 각각 별도 고정비 설정
- 월별 오버라이드 가능

### 4-6. 감가상각 관리

| 구분 | 대상 | 상각 기간 |
|------|------|-----------|
| **장비** | 에어소프트건류 (재고 아닌 장비로 분류) | **4년** |
| **시설물** | 인테리어, 책상, PC, 파르트렉 등 | **10년** |

- 자산 등록 → 월별 감가상각비 자동 계산 → 비용 반영
- **핵심**: 에어소프트건을 재고가 아닌 장비로 잡으면 절세 효과 큼

### 4-7. 부가세 시뮬레이션

- 개인/법인 각각 분기별 부가세 예상
- 매출 부가세 - 매입 부가세(공제 가능분만) = 예상 납부액
- **중고 개인매입 비중이 높을수록 부가세 부담 증가** → 경고

### 4-8. 세금 적립 안내
- 매월 희망 세액 → "세금 적립 통장에 이체하세요" 배너
- 연간 누적 적립액

## 7. UX 원칙

1. **한 화면에 입력 최소화** — 카드 전표 보고 금액만 넣으면 끝
2. **계정과목마다 쉬운 예시** — "변기 뚫었어 → 수선비", "인스타 광고 → 광고선전비"
3. **월 1회 정리** — 매일 안 해도 되게 설계
4. **역산 방식이 메인** — "세금 얼마 내고 싶어?"가 가장 눈에 띄게
5. **색상 경고** — 비용 부족(빨강), 달성률 높음(초록), 부가세 위험(주황)
6. **고정비 자동** — 🔒 표시, 연초 1회 설정
7. **모바일 대응** — 반응형 디자인 (폰으로도 확인 가능)
8. **중고매입 리스트** — 기존 습관(구매리스트 작성)을 그대로 디지털화

## 8. 추가 고려사항 (통화 조언 반영)

### 감가상각 vs 재고 — 핵심 절세
> "재고가 2억으로 잡혀 있으면 다 파는 순간 법인세가 2억으로 나옵니다"
- 에어소프트건을 **재고 아닌 장비(4년 감가상각)**로 분류 권장

### 급여로 끝전 맞추기
> "안 맞는 것들은 다 너한테 상여 때려"
- 비용 부족분이 소액일 때 → "대표 상여금 처리 검토" 팁 표시

### 법인카드 최대 활용
> "밥 먹고 술 먹고 하잖아. 무조건 법인카드 써"

### 당근페이 활용
- 소상공인 수수료 8% 미만
- 수수료 자체는 부가세 인정, 매입 물품은 불인정

### 현금 매입 증빙 관리
- 현금으로 중고 매입 시 → **간이영수증 + 상대방 신분증** 확보 필요

### 세무사 협업
- JSON 내보내기 → 세무사에게 전달 가능
- 월별 요약 인쇄 기능 → 회계사무실 제출용

## 9. 기술 구현 스펙 (Claude Code용)

### 기술 스택
- **React (JSX 싱글파일)** — Tailwind CSS
- **데이터**: JSON export/import (localStorage 사용 불가)
- **차트**: recharts (달성률 바, 부가세 파이차트, 월별 추이)
- **아이콘**: lucide-react

### 역산 로직

#### 법인세 역산
희망세액 입력 → 과세소득 = 희망세액 / 0.10 (2억 이하 10%) → 필요비용 = 총매출 - 과세소득 → 부족분 = 필요비용 - 현재비용합계

#### 부가세 역산
매출부가세 = 총매출 × 10/110
매입부가세 = (부가세공제 가능 비용 합계) × 10/110
※ 공제 가능: 임대료, 원재료, 사업자매입, 접대비, 차량, 광고, 사무용품, 통신비 등
※ 공제 불가: 급여, 4대보험, 개인매입, 현금매입(증빙없음), 감가상각비
예상납부액 = 매출부가세 - 매입부가세
